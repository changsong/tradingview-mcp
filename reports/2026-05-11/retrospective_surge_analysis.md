## 三、5只未检测到的股票 — 排除原因分析

这5只股票（300661、002281、002865、603986、002371）在5月10日被 SQZMOM v27 的6条件过滤器完全排除。

**SQZMOM v27 6条件过滤器：**
```
trendUp=true AND validCompression=true AND volOK=true AND rsiOK=true AND notHighVol=true AND signal=LONG
```

**历史上看，`validCompression`（有效压缩）是最常见的瓶颈条件。** 在5月10日的扫描中，225只股票中只有约30只满足全部6个条件。5月11日的扫描更是出现了0只合格的情况（大量 `validCompression:false`）。

这5只股票的情况可能是：
- **validCompression=false**：Squeeze 尚未形成有效压缩，或压缩已释放完毕进入趋势延伸阶段。后者意味着这些股票可能已经"提前突破"，SQZMOM 的入场信号晚了一步
- **rsiOK=false**：RSI 可能已经过热（>70 在 SQZMOM 内部逻辑中视为不合格）
- **notHighVol=false**：波动率过高，被策略过滤

> **改进建议：对"已突破但未压缩"的股票增加二级筛选通道。**
> 当前严格6条件 AND 过滤会丢失那些已经突破、处于趋势加速阶段的股票。

## 四、权重调整建议

### 调整 1：新增「全周期 ADX 强度」加分项（+8）

**当前状态：** 综合评分公式中，ADX 仅作为技术面内部的一个子项，且只在各周期独立评分，没有跨周期综合奖励。

**建议：** 在 Combined 公式中增加一个加分项：

```
+ (ALL_TF_ADX_ABOVE_30 ? +8 : 0)
```

定义：当 1W/1D/4H/1H 四个周期的 ADX 全部 > 30 时，额外 +8 分。

**理由：** 
- 这是3只大涨股最一致的公共特征（100%满足）
- 排名前4的非大涨股中，0/4 满足此条件
- 回测效果：301308 26.3→34.3（提升C→B级），600236 25.9→33.9（提升C→B级），002156 28.4→36.4（维持A级）

### 调整 2：新增「RSI 全周期持续性」加分项（+5）

**当前状态：** 各周期 RSI 独立评分，没有跨周期持续性奖励。

**建议：** 在 Combined 公式中增加：

```
+ (ALL_TF_RSI_60_75 ? +5 : 0)
```

定义：当所有周期 RSI 都在 60-75 区间时，额外 +5 分。

**理由：**
- 3只大涨股中 3/3 满足（100%），非大涨股中仅部分满足
- 60-75 代表"受控强势"——既不是弱势反弹也不是极端超买
- RSI 全周期 > 75 反而应该警惕（衰竭信号），这个加分项天然区分了"健康强势"和"过热"

### 调整 3：加大 RS 相对强度权重（技术分内部）

**当前状态：** RS 在技术评分中权重偏低。301308 的 RS=51.6（全表最高）但 tech_score 仅 29.9。

**建议：** 在技术分析评分中，增加 RS 的贡献：

```
当前: RS 作为趋势确认的辅助项（大概贡献 0-5 分）
建议: RS 直接贡献 0-15 分：
  RS > 50  → +15
  RS > 30  → +10
  RS > 15  → +5
  RS > 0   → +2
```

**理由：** RS 是经过基准（沪深300）校准后的真实超额强度，过滤了市场贝塔的影响。301308 RS=51.6 意味着它在5月10日已经大幅跑赢市场，这是最直接的"聪明钱"信号。

### 调整 4：过热惩罚梯级化

**当前状态：** 二元逻辑——过热则 -5（非趋势类），不过热不扣。

**问题：** 301308 和 002156 被标记为"过热追涨"，但实际上它们有全周期 ADX/RSI 支撑，是"强势趋势中的正常回调/加速"，而非"无根基的情绪泡沫"。

**建议：** 改为梯级惩罚：

```
过热 + 无趋势支撑 + ADX任一周期<20  → -8 （真正危险的过热）
过热 + 有趋势支撑 + 全周期ADX>30   → -2 （强势趋势中的暂时过热）
过热 + 有趋势支撑 + ADX部分>30     → -5 （默认过热）
```

**理由：** 当前公式对 301308 和 002156 的惩罚过度（它们技术面实际上非常健康），梯级化可以让"有基本面支撑的过热"保留在A/B级。

### 调整 5：SQZMOM 扫描增加"趋势延伸"二级通道

**当前状态：** `scan_stocks.js` 第348行的过滤器是严格的6条件AND：

```js
r.trendUp && r.validCompression && r.volOK && r.rsiOK && r.notHighVol && r.signal === 'LONG'
```

**问题：** 这会导致已经突破压缩阶段、进入趋势加速的股票被排除。5月11日的大涨股中有5只完全被此过滤器排除。

**建议：** 增加 OR 条件——允许 `validCompression=false` 但满足其他5条件 + 日线 ADX > 35 的股票通过：

```js
const primaryGate = r.trendUp && r.validCompression && r.volOK && r.rsiOK && r.notHighVol && r.signal === 'LONG';
const trendExtensionGate = r.trendUp && r.volOK && r.rsiOK && r.notHighVol && r.signal === 'LONG';
// trendExtensionGate 需要额外在技术中验证 ADX > 35
```
