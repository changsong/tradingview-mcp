### 2. 技术方法

```
OPT1: 压缩检测
  当前: lowerBB < lowerKC AND upperBB > upperKC (完全包含)
  改进: bb_width < (kc_width * 0.8) (宽度比率)
  原理: 压缩 = 波动率下降，用宽度比率直接测量

OPT2: 成交量
  当前: volume > SMA20 * 1.2 (高倍数)
  改进: (volume > SMA20 * 1.0) OR (volume > SMA20 * 0.9 AND sqz AND trend)
  原理: 主条件宽松，备选条件有其他确认补偿

OPT3: 评分逻辑
  当前: AND逻辑，全部条件必须满足
  改进: 评分>=60，允许部分条件失败
  原理: 灵活的权衡，强势条件补偿弱势条件

OPT4: 评分权重
  当前: sqz(30) + validComp(15) + crossover(20) + vol(15) + trend(10) + fake(10)
  改进: sqz(25) + trend(25) + momentum(25) + vol(15) + fake(10), minScore=55
  原理: 降低压缩依赖，提升趋势和动量权重

OPT5: 多时间框架
  当前: 仅日线
  改进: 日线+4小时，4H压缩可作为替代
  原理: 更高频的压缩信号
```

  - 可能增加假信号 (假压缩释放)
  - 需要其他条件补偿 (trend, momentum)

## 快速检查: 何时需要停止并重新评估

```
🛑 STOP 如果:
  1. Phase 1 后信号反而减少 (2→1)
     → 压缩倍数太严格，改为 0.85 或 0.9

  2. 回撤 >30% 或最大连续亏损 >3笔
     → 风险管理失效，检查止损设置

  3. 胜率 <40% (远低于预期 >48%)
     → 评分权重不适配，需要重新校准

  4. 纸面交易结果与回测偏差 >20%
     → 检查实时指标计算，排查错误

  5. 交易频率 >15笔/周 per stock (过于频繁)
     → 参数过度优化，需要紧缩条件

✓ 继续 如果:
  - Phase 1: 信号增加 >50% 且 胜率 >48%
  - Phase 2: 信号增加且 质量不下降
  - 纸面: 结果在 ±10% 内吻合
```
