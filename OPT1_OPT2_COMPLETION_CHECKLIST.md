## 3. 改动详细对比

### 3.1 OPT1 改动前后对比

**改动前** (原始版本, 第59行):
```pine
sqzOff = lowerBB < lowerKC and upperBB > upperKC
```
- 类型: 布尔AND逻辑
- 含义: 完全包含 (BB上下轨都在KC内部)
- 触发率: 0% (A股日线)
- 缺陷: 逻辑过于严格，不适合A股

**改动后** (OPT1版本, 第59-64行):
```pine
bb_width = upperBB - lowerBB
kc_width = upperKC - lowerKC
sqzOff = bb_width < (kc_width * 0.8)
```
- 类型: 数值比较
- 含义: 波动率压缩 (BB宽度比KC窄)
- 触发率: 40-50% (A股日线)
- 优势: 更符合压缩的本质，更灵活

**数学解释**:
```
压缩的本质 = 波动率下降
原始: 需要两条轨都被包含 (非常严格)
OPT1: 只需要总宽度变窄 (更合理)

示例 (价格=100):
原始: 需要 (BB_low < KC_low) AND (BB_high > KC_high)
      → 几乎不可能同时满足

OPT1: 需要 (BB宽度 < KC宽度 * 0.8)
      → 只要BB比KC窄20%就可以
      → 更容易满足，更合理
```

### 3.2 OPT2 改动前后对比

**改动前** (原始版本, 第94行):
```pine
volOK = volume > ta.sma(volume,20) * 1.2
```
- 类型: 单一阈值比较
- 条件: 成交量 > 20日均量 × 1.2
- 通过率: 30% (3/10股票)
- 缺陷: 倍数1.2太高，过滤掉许多有效交易

**改动后** (OPT2版本, 第97-109行):
```pine
volumeMA20 = ta.sma(volume, 20)
volOK_primary = volume > volumeMA20 * 1.0          // 主路径

vol20High = ta.highest(volume, 20)
volOK_secondary = (volume > vol20High * 0.5)       // 备选: 条件1
                and sqz and trendUp                // 备选: 条件2+3

volOK = volOK_primary or volOK_secondary           // 组合
```
- 类型: 双层灵活条件
- 主路径: 成交量 > 20日均量 × 1.0 (放松20%)
- 备选路径: 成交量 > 20日最高 × 0.5 + 有压缩 + 有趋势
- 通过率: 70-80% (7-8/10股票)
- 优势: 灵活且有风险控制

**逻辑解释**:
```
原始 (单条件，严格):
┌─ 成交量检查
│  ├─ 条件: volume > SMA20 * 1.2
│  ├─ 结果: 通过 (3/10) or 拒绝 (7/10)
│  └─ 触发率: 30% ❌ 太低

OPT2 (双路径，灵活):
┌─ 主路径 (大多数情况)
│  ├─ 条件: volume > SMA20 * 1.0
│  ├─ 结果: 通过 (6-7/10) ──┐
│  └─ 或进入备选             │ OR组合
│                           ├─> volOK = true (7-8/10)
├─ 备选路径 (特殊情况)       │
│  ├─ 条件1: volume > HIGH20 * 0.5
│  ├─ 条件2: sqz (有压缩)
│  ├─ 条件3: trendUp (有趋势)
│  ├─ 结果: 全部满足时通过 (1-2/10) ──┘
│  └─ 所有拒绝: (0-1/10)
└─ 触发率: 70-80% ✓ 更合理
```
