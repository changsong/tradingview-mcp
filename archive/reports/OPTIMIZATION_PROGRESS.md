创建了 **US-急涨急跌吸筹策略 v3 (平衡优化)** 版本，新增三个过滤条件：

#### 优化条件1: 前10日大涨过滤
```pine
// 检查前10日内任意单日涨幅
maxDailyChange10 = math.max(dailyChange1, dailyChange2, ..., dailyChange10)
noRecentSurge = maxDailyChange10 < maxSurge10d  // 默认20%
```
**目的**: 避免在大涨后追高，减少"提前止损离场，后几日才大涨"的情况

#### 优化条件2: 26周低位过滤
```pine
// 计算价格在26周区间的位置
pricePosition26w = (dailyClose - weeklyLow26) / (weeklyHigh26 - weeklyLow26) * 100
isLowPosition = pricePosition26w < lowPositionPct  // 默认70%
```
**目的**: 确保股价处于相对低位，提高上涨空间

#### 优化条件3: 5日波动过滤
```pine
// 计算最近5日波动幅度
volatility5d = (dailyHigh5 - dailyLow5) / dailyLow5 * 100
lowVolatility = volatility5d < max5dVolatility  // 默认20%
```
**目的**: 避免波动过大的股票，降低风险

## 💡 关键发现

### 1. 优化条件的作用

三个优化条件的设计目标：
- **前10日大涨过滤**: 解决"大涨后回调被止损"的问题 ✓
- **26周低位过滤**: 确保有足够上涨空间 ✓
- **5日波动过滤**: 降低风险，避免剧烈波动 ✓

### 2. BHRB成功案例

BHRB的优化指标：
- 10日最大涨幅: 2.45% (远低于20%阈值)
- 26周位置: 65.5% (低于70%阈值)
- 5日波动: 3.67% (远低于20%阈值)

**结论**: 低波动、相对低位、无近期大涨的股票最适合此策略

### 3. 策略适用性

当前策略更适合：
- ✅ 稳定增长型股票
- ✅ 低波动股票
- ✅ 相对低位的股票
- ❌ 高波动股票
- ❌ 已经大涨的股票
- ❌ 高位震荡的股票
