## OPT1 对比 - 压缩检测方法变更

### 问题分析

**原始方法的限制**:
```
压缩条件 = (BB下轨 < KC下轨) AND (BB上轨 > KC上轨)
```
- 这是"完全包含"定义
- 需要BB的两条轨道都在KC内部
- A股实际: 0/10股票 (0%触发率) ❌

**原因**: A股日线波动相对稳定，很少出现一条波动带完全包含另一条的情况

---

### OPT1 实现详解

| 方面 | 原始 | OPT1 | 说明 |
|------|------|------|------|
| **检测基础** | 轨道位置 | 轨道宽度 | 从空间关系→宽度比较 |
| **逻辑** | `and` 并联 | 单一比较 | 从2个条件→1个条件 |
| **触发条件** | 完全包含 | BB比KC窄20% | 检测波动率压缩 |
| **参数** | 无 | 0.8倍数 | 可调的灵活性 |
| **预期触发** | 0% | 40-50% | 显著提升 |

### 代码对比

#### 原始代码 (第59行)
```pine
sqzOff = lowerBB < lowerKC and upperBB > upperKC
```

#### OPT1 代码 (第59-64行)
```pine
bb_width = upperBB - lowerBB        // BB的总宽度
kc_width = upperKC - lowerKC        // KC的总宽度
sqzOff = bb_width < (kc_width * 0.8)  // BB宽 < KC宽 * 80%
```

### 逻辑解释

```
原始:
  条件1: BB下轨 < KC下轨?  ─┐
                           ├─> 必须都满足才算压缩
  条件2: BB上轨 > KC上轨?  ─┘
  触发率: 0% (太严格)

OPT1:
  BB宽度 = 上轨 - 下轨
  KC宽度 = 上轨 - 下轨
  条件: BB宽 < KC宽 * 0.8 ?  ─> 一个数学比较
  触发率: 40-50% (更合理)
```

## OPT2 对比 - 成交量灵活性

### 问题分析

**原始方法的限制**:
```
成交量条件 = 今日成交量 > 20日平均成交量 * 1.2
```
- 要求成交量比平均值高20%
- A股实际: 3/10股票 (30%触发率) ❌

**原因**: 1.2倍要求太高，导致许多有效交易被过滤

---

### OPT2 实现详解

| 方面 | 原始 | OPT2 | 说明 |
|------|------|------|------|
| **主条件倍数** | 1.2x | 1.0x | 放松 20% |
| **备选条件** | 无 | 0.5x + sqz + trendUp | 增加灵活路径 |
| **逻辑组合** | 单一 | OR (任一满足) | 从严格→灵活 |
| **通过率** | 30% | 70-80% | 大幅提升 |
| **风险控制** | 简单 | 分级 | 主路径严+备选宽 |

### 代码对比

#### 原始代码 (第94行)
```pine
volOK = volume > ta.sma(volume,20) * 1.2
```

#### OPT2 代码 (第97-109行)
```pine
volumeMA20 = ta.sma(volume, 20)

// 主路径: 放松到1.0倍
volOK_primary = volume > volumeMA20 * 1.0

// 备选路径: 低成交量但有其他确认
vol20High = ta.highest(volume, 20)
volOK_secondary = (volume > vol20High * 0.5) and sqz and trendUp

// 最终: OR组合，更灵活
volOK = volOK_primary or volOK_secondary
```

### 逻辑解释

```
原始 (单条件):
  ┌─ 成交量 > 20日均量 * 1.2 ?
  │  ✓: 通过 (只有3/10)
  └─ ✗: 拒绝 (其他7/10)
  通过率: 30% ❌

OPT2 (双路径):
  ┌─ 主路径: 成交量 > 20日均量 * 1.0 ?
  │  ✓: 通过 (约6-7/10) ──┐
  │  ✗: 进入备选          │
  │                      ├─> volOK = true
  ├─ 备选: 成交量 > 20日最高 * 0.5 AND 压缩 AND 上升趋势 ?
  │  ✓: 通过 (约1-2/10) ──┘
  │  ✗: 拒绝
  └─ 拒绝 (仅0-1/10)
  通过率: 70-80% ✓
```
